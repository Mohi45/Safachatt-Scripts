Object

An object is a real-world entity that has state and behavior. It is an instance of a class.

Also an Object in Java is an instance of a class that contains data (variables) and behavior (methods).

Syntax to Create an Object

ClassName objectName = new ClassName();

class Student {

    String name;
    int age;

    void display() {
        System.out.println("Name: " + name);
        System.out.println("Age: " + age);
    }

}

public class Main {
    public static void main(String[] args) {

        Student s1 = new Student(); // object creation

        s1.name = "Rahul";
        s1.age = 20;

        s1.display();
    }
}

Types of Object Creation in Java

1. Using new Keyword (Most Common) Student s1 = new Student();
2. Using newInstance() Student s1 = Student.class.newInstance();
3. Using clone() Student s2 = (Student) s1.clone();

Inheritance

Inheritance is an important concept of Object-Oriented Programming (OOP). It allows one class to acquire the properties and methods of another class.

class Animal {

    void eat() {
        System.out.println("Animal is eating");
    }

}

class Dog extends Animal {

    void bark() {
        System.out.println("Dog is barking");
    }

}

public class Main {

    public static void main(String[] args) {

        Dog d = new Dog();

        d.eat();   // inherited method
        d.bark();  // own method

    }

}

Why Inheritance is Used

  • Code Reusability
  • Reducing redundancy
  • Improving maintainability
  • Implementing hierarchical relationships

    • Types of Inheritance in Java

    Java supports 5 types of inheritance conceptually, but 4 are implemented using classes.

  • Single Inheritance
  • MultiLevel Inheritance
  • Hierarchical Inheritance
  • Multiple Inheritance
  • Hybrid Inheritance

    • Single Inheritance

    Single inheritance is a type of inheritance where one child class inherits properties and methods from one parent class.

    Explained in Hindi

    Single Inheritance ka matlab hai ki ek child class sirf ek hi parent class se properties aur methods inherit karti hai.

    class Account {

    void deposit() {
        System.out.println("Money deposited");
    }

}

class SavingsAccount extends Account {
    void interest() {
        System.out.println("Interest added");
    }

}

public class Main {
    public static void main(String[] args) {
        SavingsAccount s = new SavingsAccount();
        s.deposit();
        s.interest();

    }

}

    Advantages of Single Inheritance

    ✔ Code reuse hota hai
    ✔ Code short ho jata hai
    ✔ Maintain karna easy hota hai
    ✔ Real world relationship represent karta hai

    Multilevel Inheritance

    Multilevel inheritance is a type of inheritance where a class inherits from another class, which itself inherits from another class.

    Explained in Hindi

    Multilevel Inheritance me ek class dusri class ko inherit karti hai aur fir third class us second class ko inherit karti hai.

    class Animal {

    void eat() {
        System.out.println("Animal is eating");
    }

}

class Dog extends Animal {
    void bark() {
        System.out.println("Dog is barking");
    }

}

class Puppy extends Dog {
    void weep() {
        System.out.println("Puppy is weeping");
    }

}

public class Main {
    public static void main(String[] args) {
        Puppy p = new Puppy();
        p.eat();   // from Animal
        p.bark();  // from Dog
        p.weep();  // from Puppy

    }

}

    Hierarchical Inheritance

    Hierarchical inheritance is a type of inheritance where multiple child classes inherit from a single parent class.

    Explained in Hindi

    Hierarchical Inheritance me ek parent class se multiple child classes inherit karti hain.

    One Parent → Many Child Classes

    class Animal {

    void eat() {
        System.out.println("Animal is eating");
    }

}

class Dog extends Animal {
    void bark() {
        System.out.println("Dog is barking");
    }

}

class Cat extends Animal {
    void meow() {
        System.out.println("Cat is meowing");
    }

}

public class Main {

    public static void main(String[] args) {
        Dog d = new Dog();
        Cat c = new Cat();

        d.eat();  
        d.bark();

        c.eat();  
        c.meow();

    }

}

    Multiple Inheritance

    Multiple inheritance is a type of inheritance where one class inherits properties and methods from more than one parent class. In Java, it is supported using interfaces but not with classes.

    Explained in Hindi

    Multiple Inheritance ka matlab hai ki ek class ek se zyada parent classes se properties aur methods inherit kare.

    One Child → Many Parent Classes

    class A {
    void show() {
        System.out.println("Class A");
    }
}

class B {
    void display() {
        System.out.println("Class B");
    }
}

class C extends A, B { }   // ❌ Not allowed in Java

    Reason – Diamond Problem

    Agar do parent classes me same method ho, to child class confuse ho jati hai ki kaunsa method use kare.

    Multiple Inheritance Using Interfaces

    interface Animal {

    void eat();

}

interface Bird {

    void fly();

}

class Bat implements Animal, Bird {

    public void eat() {
        System.out.println("Bat eats fruits");
    }

    public void fly() {
        System.out.println("Bat can fly");
    }

}

public class Main {

    public static void main(String[] args) {

        Bat b = new Bat();

        b.eat();
        b.fly();

    }

}

    Hybrid Inheritance

    Hybrid inheritance is a combination of two or more types of inheritance. In Java it is implemented using interfaces because multiple inheritance through classes is not supported.

    Explained in Hindi

    ⚠️ Java me Hybrid inheritance directly classes ke through possible nahi hai kyunki multiple inheritance classes me allowed nahi hai. Lekin interfaces ka use karke Hybrid inheritance achieve kiya ja sakta hai.

    interface Camera {

    void takePhoto();

}

interface Phone {

    void call();

}

class Device {

    void powerOn() {
        System.out.println("Device is powered on");
    }

}

class Smartphone extends Device implements Camera, Phone {

    public void takePhoto() {
        System.out.println("Taking photo");
    }

    public void call() {
        System.out.println("Calling someone");
    }

}

    Output

    Device is powered on
    Taking photo
    Calling someone

    Advantages

    ✔ Flexible design
    ✔ Code reuse
    ✔ Multiple features ek class me
    ✔ Complex systems build karne me useful

    Disadvantages

    ❌ Code complexity increase ho sakti hai
    ❌ Debugging difficult ho sakti hai

    Real Life Example

    Vehicle (Parent Class)
    Common properties:
    start()
    stop()
    speed
    Car (Child Class)
    airCondition()
    Bike (Child Class)
    kickStart()
    Dono Vehicle ke methods use kar sakte hain.

    Polymorphism

    Polymorphism is the ability of a method or object to take many forms. In Java it is achieved using method overloading and method overriding.

    Explained in Hindi

    Polymorphism ka meaning hai:
    “One name, many forms”
    Matlab ek method ya function different ways me behave kar sakta hai.

    Compile Time Polymorphism (Method Overloading)

    Jab same method name ho lekin parameters different ho, to use method overloading kehte hain.

    class MathOperation {

    int add(int a, int b) {
        return a + b;
    }

    int add(int a, int b, int c) {
        return a + b + c;
    }

}

public class Main {

    public static void main(String[] args) {

        MathOperation m = new MathOperation();

        System.out.println(m.add(5, 10));
        System.out.println(m.add(5, 10, 15));

    }

}

    Run Time Polymorphism (Method Overriding)

    Jab child class parent class ke method ko redefine karti hai, use method overriding kehte hain.

    class Animal {
    void sound() {
        System.out.println("Animal makes sound");
    }

}

class Dog extends Animal {
    void sound() {
        System.out.println("Dog barks");
    }

}
public class Main {

    public static void main(String[] args) {
        Animal a = new Dog();
        a.sound();

    }

}

    Abstraction

    Abstraction is the process of hiding implementation details and showing only essential functionality to the user. In Java it is achieved using abstract classes and interfaces.

    Explained in Hindi

    Abstraction ka matlab hai:
    Important information ko show karna aur unnecessary details ko hide karna.
    Simple words me:
    User ko sirf functionality dikhai deti hai
    Internal working hide hoti hai

    Real Life Example

    Jab aap ATM use karte ho:
    Card insert
    PIN enter
    Money withdraw
    Lekin ATM ka internal process (bank server communication, verification) user ko nahi dikhta.
    Yahi Abstraction hai. 

    abstract class Animal {

    abstract void sound();

    void eat() {
        System.out.println("Animal is eating");
    }

}

class Dog extends Animal {

    void sound() {
        System.out.println("Dog barks");
    }

}

public class Main {

    public static void main(String[] args) {

        Dog d = new Dog();

        d.sound();
        d.eat();

    }

}

    Advantages of Abstraction

    ✔ Code security
    ✔ Complexity reduce hoti hai
    ✔ Code maintain karna easy
    ✔ Real world modeling possible

    Encapsulation

    Encapsulation is the process of wrapping data and methods into a single unit and restricting direct access to data using private variables.

    Explained in Hindi

    Encapsulation ka matlab hai:
    Data (variables) aur methods ko ek single unit (class) me wrap karna aur data ko direct access se protect karna.
    Simple words me:
    Data ko hide karna
    Access sirf methods ke through dena
    Isko Data Hiding bhi kehte hain.

    Real Life Example

    Example: Bank Account
    Bank account me:
    Balance → private hota hai
    Direct access nahi hota
    User balance change nahi kar sakta directly.
    User sirf methods use karta hai:
    deposit()
    withdraw()
    Ye hi Encapsulation hai. 

    class Student {

    private String name;

    public void setName(String n) {
        name = n;
    }

    public String getName() {
        return name;
    }

}
public class Main {

    public static void main(String[] args) {

        Student s = new Student();

        s.setName("Mohit");

        System.out.println(s.getName());

    }

}

    Encapsulation Kaise Achieve Karte Hain

    Java me encapsulation 3 steps se achieve hota hai:
    1️⃣ Variables ko private declare karna
    2️⃣ Getter method banana (data read karne ke liye)
    3️⃣ Setter method banana (data update karne ke liye)

    Interface

    An interface in Java is a collection of abstract methods that a class must implement. It is used to achieve abstraction and multiple inheritance.

    Explained in Hindi

    Interface Java me ek blueprint hota hai jo class ko batata hai ki kaun-kaun se methods implement karne hain.
    Simple words me:
    Interface ek contract hota hai jo class ko follow karna padta hai.
    Interface me methods declare hote hain, lekin unki body nahi hoti (normally).

    Syntax of Interface

    interface InterfaceName {
    void method1();
    void method2();

}

    Example: 

    interface Animal {

    void sound();

}

class Dog implements Animal {
    public void sound() {
        System.out.println("Dog barks");
    }

}

public class Main {

    public static void main(String[] args) {
        Dog d = new Dog();
        d.sound();

    }

}

    Advantages of Interface

    ✔ Multiple inheritance support
    ✔ Loose coupling
    ✔ Better code design
    ✔ Abstraction achieve karta hai

    Real Life Example

    Remote ek interface ki tarah hota hai.
    Buttons:
    power()
    volumeUp()
    volumeDown()
    Different devices:
    TV
    AC
    Music System
    Sab devices same interface follow karte hain.
